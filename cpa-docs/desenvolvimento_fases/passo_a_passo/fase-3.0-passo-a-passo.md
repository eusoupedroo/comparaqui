# Passo a passo - Fase 3.0 - Produto com backend

## Objetivo

Implementar uma aplicação real com frontend, backend, autenticação, persistência e APIs, somente após as fases preparatórias.

## Etapa 1 - Confirmar pré-requisitos

### Descrição

Validar se o projeto está pronto para deixar de ser apenas mockup.

### Ações necessárias

- Confirmar conclusão da fase 1.1.
- Confirmar conclusão da fase 1.2.
- Confirmar conclusão da fase 2.0.
- Revisar contratos de API planejados.
- Revisar regras de domínio formalizadas.

### Arquivos envolvidos

- `cpa-docs/desenvolvimento_fases/fase-1.1-consolidacao-do-mockup.md`
- `cpa-docs/desenvolvimento_fases/fase-1.2-dominio-mockado-forte.md`
- `cpa-docs/desenvolvimento_fases/fase-2.0-preparacao-next-nest.md`

### Critério de conclusão

Não há pendências críticas impedindo backend e persistência.

## Etapa 2 - Criar base técnica

### Descrição

Criar as aplicações Next.js e NestJS de acordo com o plano técnico.

### Ações necessárias

- Criar frontend Next.js.
- Criar backend NestJS.
- Configurar banco de dados.
- Configurar autenticação.
- Criar estrutura inicial de módulos.

### Arquivos envolvidos

- Nova estrutura futura de frontend.
- Nova estrutura futura de backend.
- Configurações de ambiente.

### Critério de conclusão

As aplicações sobem localmente e possuem integração básica planejada.

## Etapa 3 - Implementar domínio principal

### Descrição

Levar para o backend as regras de lista, produtos, mercados, ofertas, similares, substituições e comparação.

### Ações necessárias

- Implementar persistência de listas.
- Implementar cadastro e consulta de produtos.
- Implementar cadastro e consulta de mercados.
- Implementar ofertas por mercado.
- Implementar comparação via API.
- Implementar regras de similaridade e substituição.

### Arquivos envolvidos

- Módulos backend planejados.
- Componentes e páginas frontend futuras.
- Contratos de API.

### Critério de conclusão

O fluxo principal funciona com dados persistidos e APIs reais.

## Checklist final da fase

- [ ] Pré-requisitos concluídos.
- [ ] Next.js criado.
- [ ] NestJS criado.
- [ ] Banco configurado.
- [ ] Autenticação básica implementada.
- [ ] Listas persistidas.
- [ ] Produtos, mercados e ofertas disponíveis via API.
- [ ] Comparação executada no backend.
- [ ] Regras de similares e substituições no backend.
- [ ] API versionada.
